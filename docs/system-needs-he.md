# דרישות מערכת מפורטות למולטי-דב VPS

מסמך זה מרכז את צרכי המערכת והיכולות שהפורטל והשרתים צריכים לספק בפועל.
המטרה היא לשמש מסמך דרישות תפעולי, לא רק תיאור רעיוני.

## 1. פורטל ניהול מרכזי

- נדרש דף בית אחיד לכל השרתים שמציג מצב כללי של המערכת.
- נדרש מסך ניהול `/manage/` שמרכז את כל כלי התפעול במקום אחד.
- נדרש להציג גרסת מערכת, זמן build אחרון, ומצב שירותים פעילים.
- נדרש שתהיה תאימות מלאה לטעינה דרך דפדפן רגיל ללא כלים חיצוניים.
- נדרש ממשק נוח גם במסך קטן וגם במסך גדול.
- נדרש רענון מצב מהיר בלי טעינה מחדש של כל השרת.

## 2. ניהול פרויקטים

- נדרש רישום פרויקטים שנפרסים על אותו VPS.
- נדרש יכולת להוסיף פרויקט חדש מתוך מאגר GitHub.
- נדרש לעדכן פרויקט קיים מתוך `main` או ענף אחר לפי צורך.
- נדרש להפעיל, לעצור, ולאתחל פרויקט דרך הפורטל.
- נדרש להציג לוגים של כל פרויקט בצורה נוחה.
- נדרש לזהות תהליך PM2 נכון לכל פרויקט.
- נדרש תמיכה ביישומים שמפוצלים ל-`server/`, `client/`, או `dashboard/`.

## 3. פריסת קוד ובילד

- נדרש שהמערכת תוכל למשוך קוד חדש מ-GitHub ולבנות אותו מהמקור.
- נדרש שה-build יתבצע מה-`main` המעודכן ולא מגירסה ישנה שנשארה בזיכרון.
- נדרש restart אחרי build מוצלח.
- נדרש תמיכה ב-`npm install` רגיל וב-`npm --prefix` עבור תתי-תיקיות.
- נדרש שהמערכת תדע לזהות קבצי `package.json` נוספים ולבנות אותם לפי הצורך.
- נדרש לשמור תאימות בין build, סביבת ריצה, ו-nginx.

## 4. ניהול TLS ו-SSL

- נדרש מסך לניהול תעודות SSL ו-TLS.
- נדרש תמיכה בתעודות לפי דומיין ראשי ולפי דומיינים של פרויקטים.
- נדרש fallback ברור כאשר אין תעודה ייעודית.
- נדרש שהמערכת תוכל לשמור תעודות מותאמות ידנית.
- נדרש שהממשק יראה אילו דומיינים פעילים ואילו לא.
- נדרש להבדיל בין תעודה מערכתית לבין תעודה של פרויקט.

## 5. ניהול DB

- נדרש ניהול של מכונות MySQL מקומיות ומרוחקות.
- נדרש להגדיר מאגר DB לכל פרויקט בעת התקנה.
- נדרש תמיכה בהעברת פרויקט ל-DB machine אחרת.
- נדרש יצירת משתמש, סיסמה והרשאות באופן אוטומטי.
- נדרש bootstrap של הרשאות root דרך SSH כאשר ה-DB מרוחק.
- נדרש בדיקת חיבור לפני החלפת מכונת DB.
- נדרש שהפורטל יראה איזו מכונת DB מחוברת לכל פרויקט.

## 6. מפת דומיינים ו-Nginx

- נדרש מיפוי אוטומטי של דומיינים לשרתים ולפרויקטים.
- נדרש יצירה ועדכון של nginx בלי התערבות ידנית בכל שינוי.
- נדרש תמיכה בדומיין ראשי וב-aliases.
- נדרש שהמערכת תדע לשחזר nginx אחרי שינויי deploy.
- נדרש להגן על דומיינים לפי צורך עם הרשאות מתאימות.

## 7. SSH ומפתחות

- נדרש מסך לניהול מפתחות SSH.
- נדרש יצירה אוטומטית של הגדרות `ssh config` לפי GitHub user.
- נדרש שהמערכת תוכל לשייך מפתח נכון לפי מאגר GitHub.
- נדרש שימוש ב-root כשצריך, ובמשתמש sudo כשיש.
- נדרש להראות למשתמש איזה key משמש לפרויקט.

## 8. שירות Proxy מקומי

- נדרש שרת Proxy מקומי על ה-VPS עצמו.
- נדרש אפשרות להפעיל ולכבות את שירות ה-proxy מהפורטל.
- נדרש להגדיר:
  - כתובת listen
  - port
  - הרשאות גישה
  - username
  - password
  - רשימת ports מותרים ל-CONNECT
- נדרש שה-proxy יתאים לשימוש בדפדפן וב-`curl`.
- נדרש שיפור UX כך שהמשתמש יוכל להעתיק URL ופרטי התחברות בלחיצה.
- נדרש להציג את ה-proxy credentials ישירות במסך `/manage/proxy/`.
- נדרש כפתור `Copy credentials` שעובד גם אם Clipboard API לא זמין.

## 9. בדיקת Proxy

- נדרש כפתור בדיקה ל-proxy הנבחר.
- נדרש שהבדיקה תחזיר את ה-IP האמיתי שה-proxy מוציא דרכו.
- נדרש שהבדיקה תבדוק גם את המיקום הגיאוגרפי של ה-IP.
- נדרש להציג עיר, מדינה, ארגון, ואזור זמן כאשר המידע זמין.
- נדרש להבחין בין proxy עובד לבין proxy שלא עבר בדיקה.
- נדרש לעדכן את מצב ה-proxy אחרי כל בדיקה.

## 10. ייבוא JSON של Proxies

- נדרש אפשרות להעלות קובץ JSON עם רשימת proxies.
- נדרש ולידציה של מבנה ה-JSON לפני שמייבאים.
- נדרש תמיכה בפורמטים נפוצים:
  - מערך פשוט
  - `proxies`
  - `entries`
  - `proxy_pool.entries`
- נדרש לבדוק כל proxy אחרי הייבוא.
- נדרש לבחור proxy עובד ראשון באופן אוטומטי.
- נדרש לאפשר עריכה ידנית של proxy נבחר.
- נדרש לאפשר העתקה של URL ופרטי התחברות לכל proxy.

## 11. Proxy חיצוני / Egress

- נדרש תמיכה ב-proxy חיצוני או tunnel שיוצא דרך מדינה מסוימת.
- נדרש שהמערכת תוכל לשייך profile של egress לכל פרויקט.
- נדרש שהפרויקט לא ישבור את ה-nginx או את מיפוי הדומיינים כאשר ה-egress משתנה.
- נדרש שה-portal יראה את מצב ה-egress ואת היעד הגיאוגרפי שלו.

## 12. שירות VPN לפרוקסי

- נדרש תמיכה ב-VPN שמזין את ה-proxy המקומי.
- נדרש שה-VPN יוכל להביא exit IP ממדינה מוגדרת.
- נדרש לנטר את מצב ה-VPN ואת מצב השירותים שתלויים בו.
- נדרש שהמערכת תישאר יציבה גם אם ה-VPN נופל.
- נדרש שה-proxy המקומי יישאר נגיש גם כאשר ה-VPN מחובר מחדש.

## 13. אבטחה והרשאות

- נדרש Basic Auth ל-`/manage/`.
- נדרש לא לשמור סודות בתוך מסמכי תיעוד.
- נדרש לשמור סיסמאות בקבצי config מקומיים בלבד.
- נדרש להפריד בין credentials של המערכת לבין credentials של הפרוקסי.
- נדרש להציג נתונים רגישים רק בתוך ממשק הניהול המאובטח.

## 14. אמינות ותפעול

- נדרש שהמערכת תעלה מחדש אחרי reboot של השרת.
- נדרש systemd/PM2 פעילים אוטומטית.
- נדרש שהשירותים יהיו ניתנים לשחזור גם אחרי עדכון קוד.
- נדרש בדיקות smoke בסיסיות אחרי פריסה.
- נדרש לוגים ברורים כשפריסה או בדיקה נכשלים.

## 15. חוויית משתמש

- נדרש שהמשתמש יבין מיד מה מצב השרת ומה עובד כרגע.
- נדרש כפתורים ברורים לפעולות נפוצות:
  - שמירה
  - הפעלה
  - בדיקה
  - העתקה
  - ייבוא
- נדרש טקסט הסבר קצר ליד כל פעולה מסוכנת.
- נדרש שמסך ה-proxy והמסך הראשי יהיו עקביים מבחינת שפה ומבנה.

## 16. צרכים תפעוליים להמשך

- נדרש מסמך אחד שמרכז את הסטנדרט של כל VPS חדש.
- נדרש שהסטנדרט יהיה ניתן להרחבה בלי לשבור התקנות קיימות.
- נדרש לשמור תאימות בין code, config, ו-deploy בפועל.
- נדרש שהמסמך ישמש גם לצוות וגם לפריסות אוטומטיות עתידיות.
